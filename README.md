# Shotsmith — Spendaily App Store screenshot studio

Create App Store / Google Play screenshots like the live Spendaily listing: a
two-tone headline, an eyebrow with accent rules, a real device mockup with your
screenshot inside, and **floating feature pills** scattered around the phone —
all editable, all exportable at full resolution. Copy, pills, font pairings and
whole screen sets can be generated by **Kimi**, grounded in the Spendaily brand
blueprint.

## Run it

```bash
npm start          # → http://localhost:4317
```

A tiny Node server (no dependencies) serves the tool and proxies Kimi. Open the
URL, pick **Spendaily**, and you start with the full 10-screen set preloaded.

## Why a server?

The Kimi API sends no CORS headers, so the browser can't call it directly. The
server proxies `POST /api/kimi` and injects the API key **server-side**, so the
key is never shipped to or visible in the browser.

- Key is read from `KIMI_API_KEY` (env) or the gitignored `.kimi-key` file.
- Model: `kimi-for-coding` at `https://api.kimi.com/coding/v1` (temperature 1,
  OpenAI-compatible). It's a "thinking" model, so AI calls take ~15–90s.

## Using it

- **Device** — iPhone (1284 × 2778) or Android (1080 × 2340). Swaps the frame.
- **Screens** — each tab is one screenshot; add / duplicate / rename (double-click).
- **Text** — eyebrow, two-tone headline (toggle which lines use the accent colour),
  accent bar, subtitle, position.
- **Pills** — the floating badges. Add, edit, restyle (solid / glass / tint),
  add a star-rating pill, **drag them on the canvas**, or auto-arrange.
- **Style** — brand palettes (Standard, Ocean, Sunset, Forest, Rose Gold, Midnight),
  gradient / mesh / solid backgrounds.
- **Device panel** — phone scale / position / rotation / shadow, and upload the
  screenshot that goes inside the frame.
- **Fonts** — pick a display + body pairing (or let Kimi suggest one).
- **✨ AI** — Generate copy, Generate pills, Suggest fonts, or Generate a full set.
- **Export** — single PNG, or all screens as a ZIP.

## Adding another brand

Drop a `brands/<name>.js` (same shape as `brands/spendaily.js`) plus a
`brands/<name>-blueprint.md`, register it on `window.BRANDS`, and it appears in
the brand picker. Kimi is automatically grounded with that brand's blueprint.

## Frame assets

`Iphone-frame-upscaled.png` and `Android-frame.png` are the device frames. The
renderer auto-detects the screen cut-out from the frame's alpha, so swapping in a
new frame PNG generally just works.
